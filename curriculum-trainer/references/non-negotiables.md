# Program Non-Negotiables

These eight rules are the locked decisions of the program. Violating any of
them breaks the program, even if the resulting content looks polished.

When a user request conflicts with any item below, the agent must name the
conflict and ask before complying. See CLAUDE.md — Non-Negotiable Conflict
Handling.

---

## 1. The pedagogical method is Productive Failure

- Students solve a problem *before* being taught the concept.
- Each cycle follows: DO → FAIL → NAME → FIX.
- Failure must be *engineered*, not random — designed into the task so the
  intended weakness shows up clearly.
- Every failure must be followed by consolidation — the evaluator naming the
  gap and the student earning the cure. Struggle without consolidation does
  not teach.

## 2. The mental model is student-as-manager, agent-as-worker

The student is not a prompter. They are a manager who hires, trains, and
delegates to an AI worker. All content must reinforce this frame. Never write
content that puts the student back in the "press a button and hope" posture.

## 3. No folder, no templates, no meta-agent creator

- Students never receive a pre-built folder of agent documents.
- Students never receive a fill-in-the-blanks template before they have felt
  the need for what the template solves.
- Students never use a meta-agent creator that interviews them and produces the
  agent for them.
- Concepts enter through *felt problems*, never definitions.

## 4. Vocabulary follows experience, never precedes it

Students do not need to learn the terms "skill document," "system prompt,"
"context window," or "tool use." They need to hit the wall each term exists to
solve. Names may be attached afterward if useful — but never lead with the
name, and never make the name the lesson.

## 5. The evaluator is stage-aware

The evaluator never reveals a problem the curriculum hasn't reached yet. Early
on, it surfaces only the *one* dominant gap the next component fixes. It is not
a comprehensive critic. It is a patient senior colleague pointing out the next
thing the learner needs to see.

## 6. One engine, two tracks

There is one shared curriculum engine. Two tracks are built on it:
- Teen: "Build Your AI Co-Founder" — 16 weeks, four acts, venture storyline.
- Corporate: "Build Your AI Co-Worker" — 6–8 weeks, five cycles,
  real-process storyline.

Corporate content is always customised by industry context, audience level, and
process type. There is no "generic" corporate — the parameters are always set.

## 7. The corporate security cycle is taught as a felt failure

Data security and privacy in the corporate edition are not a compliance module.
They are an engineered failure cycle, taught with the same productive-failure
logic as everything else. The learner builds something that works but unsafely;
the evaluator catches it. This may never be reduced to slides or a quiz.

## 8. The final win must be real

The end of each program produces a tangible, demonstrable outcome:
- Teen: a working agent + a venture concept the student has run.
- Corporate: a working agent automating a real process + before/after time data.

"Successful completion" may never be defined by attendance, quiz scores alone,
or simulated work.
