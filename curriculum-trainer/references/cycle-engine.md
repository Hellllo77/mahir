# The Cycle Engine — Reference

The shared curriculum engine that runs both tracks. This is a knowledge file
describing the engine structure. It is not a workflow instruction.

---

## The Core Loop

Every cycle runs the same four-beat sequence:

1. **DO** — The learner does a real piece of work using AI.
2. **FAIL** — The work hits an engineered failure: a weakness designed into the
   task so it shows up clearly.
3. **NAME** — The evaluator names the gap in plain language and reframes the
   failure as a setup problem, not "AI is broken."
4. **FIX** — The learner closes the gap by adding one component to their agent,
   then sees the same work succeed.

The loop is the engine. Everything else is wrapper.

---

## The Five Failure Types and Their Cures

| Failure | What the learner sees | The cure earned |
|---|---|---|
| **Randomness** | Ask AI twice, get materially different results. Nothing holds still. | A defined, consistent way of working — the agent given a stable approach. |
| **Hallucination** | The agent confidently invents facts, statistics, or details the learner is about to act on. | Real sources — the agent grounded in documents, web access, or approved data. |
| **Skipped / improvised steps** | The agent does the same job a different way every time and forgets steps, because no procedure was written down. | A skill document — the procedure pinned in writing so the agent follows it identically. |
| **Format drift** | Output structure changes run to run; data is lost or misplaced; results are inconsistent. | Explicit output formats and templates — consistent, immediately usable results. |
| **Unsafe data handling** *(corporate only)* | The learner builds something that works — but sends or exposes real data in a way that would be a real security or privacy problem. | Safe-handling discipline — knowing what data is safe to give an agent, where inputs and outputs go, and aligning with AI-use policy. |

## Teen Track — Four Acts

| Act | Weeks | Venture state | Failure exposed | Cure earned |
|---|---|---|---|---|
| 1 — Ideation | 1–4 | Venture doesn't exist yet | Randomness (+ hallucination shock in Week 1) | Defined way of working |
| 2 — Marketing | 5–8 | Venture exists, needs customers | Hallucination (in full) | Real sources / web access |
| 3 — Operational | 9–12 | Venture is running and messy | Skipped/improvised steps | Skill document |
| 4 — Processing | 13–16 | Venture has real volume | Format drift | Output format + templates |

Weekly rhythm within each act: Week 1 = Failure. Week 2 = Fix. Week 3 = Stretch.
Week 4 = Win.

## Corporate Track — Five Cycles

| Cycle | Job given | Failure exposed | Cure earned |
|---|---|---|---|
| 1 — Consistency | Get AI to perform the chosen process for the first time | Randomness | Defined, consistent way of working |
| 2 — Real Information | Have the agent use facts, data, or documents the process needs | Hallucination | Grounded sources |
| 3 — Security Gate | The process now involves real company information | Unsafe data handling | Safe-handling discipline |
| 4 — Reliable Procedure | Agent must run the full multi-step process unattended | Skipped/improvised steps | Written procedure (skill document) |
| 5 — Reliable Output | Agent processes real volume, must produce consistent output | Format drift | Explicit output formats and templates |

Cycle pacing target: roughly 1.5 weeks per cycle across a 6–8 week program.

## The Evaluator's Role

The evaluator is the learner's substitute judgement during construction. Its
core behaviours:

- Names the gap in plain language — never just scores.
- Is stage-aware — surfaces only the one gap the next component fixes.
- Reframes failure as a setup problem: "you didn't get a bad AI — you gave it
  no instructions."
- Shifts voice in the closing phase: from patient colleague to client or
  stakeholder judging a real deliverable.

The evaluator's stage-awareness rule is non-negotiable: it never reveals a
problem the curriculum hasn't reached yet.
